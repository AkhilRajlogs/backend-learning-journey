# TreeSet in Java  
  
---  
  
## Overview  
  
TreeSet is a Set implementation that stores elements in **sorted order**.  
  
Unlike HashSet and LinkedHashSet, TreeSet maintains elements according to their **natural ordering** or a **custom comparator**.  
  
TreeSet does not allow duplicate elements.  
  
---  
  
## Key Characteristics  
   
- Stores unique elements   
- Maintains sorted order  
- TreeSet does not allow null elements because comparison is required for ordering.  
- Slightly slower than HashSet and LinkedHashSet due to tree balancing operations.  
- Implements the **SortedSet** interface  
  
---  
  
## Internal Structure  
  
TreeSet internally uses a **Red-Black Tree**, which is a type of **self-balancing binary search tree**.  
  
Properties of Red-Black Tree:  
  
- Ensures the tree remains balanced  
- Guarantees predictable performance  
- Maintains sorted ordering of elements  
  
Because of this structure, operations require **tree traversal** rather than direct hashing.  
  
---  
  
## Time Complexity  
  
Because TreeSet uses a balanced tree:  
  
- add(): O(log n)  
- remove(): O(log n)  
- contains(): O(log n)  
  
This is slower than hash-based structures but allows **ordered data retrieval**.  
  
---  
  
## Example Behavior  
  
Example:  
  
    TreeSet<Integer> numbers = new TreeSet<>();  
  
    numbers.add(50);  
    numbers.add(10);  
    numbers.add(30);  
    numbers.add(20);  
  
Output:  
  
    [10, 20, 30, 50]  
  
Elements are automatically sorted.  
  
---  
  
## HashSet vs LinkedHashSet vs TreeSet  
  
| Feature | HashSet | LinkedHashSet | TreeSet |  
|---------|---------|---------------|--------|  
| Order | Not guaranteed | Insertion order | Sorted order |  
| Internal Structure | HashMap | LinkedHashMap | Red-Black Tree |  
| Performance | O(1) average | O(1) average | O(log n) |  
    
---  
  
## When to Use TreeSet  
  
Use TreeSet when:  
  
- Elements must remain **sorted**  
- You need **range operations**  
- Ordering of data is important  
  
Examples:  
  
- Ranking systems  
- Sorted leaderboards  
- Maintaining ordered unique values  
  
---  
  
## Important Note  
  
TreeSet determines element ordering using:  
  
    Comparable  
    Comparator  
  
If elements do not implement Comparable and no Comparator is provided, a **ClassCastException** may occur.
