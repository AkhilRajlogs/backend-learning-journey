# Comparable in Java  
  
---  
  
## Overview  
  
Comparable is an interface used to define the **natural ordering of objects**.  
  
It is part of the **java.lang** package.  
  
When a class implements Comparable, it must override the **compareTo() method** to specify how objects should be compared.  
  
This allows collections like **TreeSet**, **TreeMap**, and sorting utilities to automatically order objects.  
  
---   
  
## Method Definition  
   
The Comparable interface contains a single method:  
  
    int compareTo(T other)  
  
---  
  
## How compareTo Works  
  
The method compares the current object with another object.  
  
Return values determine ordering:  
  
| Return Value | Meaning |  
|--------------|--------|  
| Negative | Current object is smaller |  
| Zero | Objects are equal |  
| Positive | Current object is greater |  
  
Example:  
  
    a.compareTo(b)  
  
---  
  
## Example with Numbers  
  
    Integer a = 10;  
    Integer b = 20;  
  
    System.out.println(a.compareTo(b));  
  
Output:  
  
    -1  
  
Because **10 < 20**.  
  
---  
  
## Why Comparable is Needed  
  
Comparable allows Java collections to **sort objects automatically**.  
   
Example:  
  
    TreeSet<Integer> numbers = new TreeSet<>();  
  
Because Integer implements Comparable, TreeSet can maintain sorted order.  
  
---  
  
## Custom Object Sorting  
  
If we create a custom class like **Employee**, Java does not know how to sort it.  
  
We must implement Comparable:  
  
    class Employee implements Comparable<Employee>  
  
Then define sorting logic inside:  
  
    compareTo()  
  
---  
  
## Example Scenario  
  
Sort employees by **salary**.  
  
    @Override  
    public int compareTo(Employee other) {  
        return this.salary - other.salary;  
    }   
  
This means employees will be sorted by salary.  
  
---  
  
## Where Comparable is Used  
  
Comparable is used by:  
  
- TreeSet  
- TreeMap  
- Collections.sort()  
- Arrays.sort()  
  
---  
  
## Limitation of Comparable  
  
Comparable allows **only one natural sorting rule**.  
  
Example:  
  
Employee may need sorting by:  
  
- Salary  
- Name  
- Age  
  
Comparable can define **only one default order**.  
  
For multiple sorting strategies we use **Comparator**.  
  
---  
  
## Comparable vs Comparator (Preview)  
  
| Feature | Comparable | Comparator |
|-------|------------|------------|
| Package | java.lang | java.util |
| Method | compareTo() | compare() |
| Sorting Logic | Inside class | Outside class |
| Sorting Rules | Single | Multiple |

---  
  
## Summary  
  
Comparable defines the **natural ordering of objects**.  

Classes implement Comparable when they have a **default logical sorting rule**.  
  
It is commonly used with **TreeSet, TreeMap, and sorting utilities**.
