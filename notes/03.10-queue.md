# Queue in Java  
  
---  
  
## Overview  
  
The **Queue interface** is part of the **java.util package** and extends the **Collection interface**.  
  
A queue represents a collection designed for **holding elements prior to processing**.  
  
Most queue implementations follow the **FIFO (First-In-First-Out)** principle.  
  
---  
  
## FIFO Principle  
  
FIFO means the **first element inserted is the first element removed**.  
  
Example sequence:  
  
Insert order:  
  
    A → B → C  
  
Removal order:  
  
    A → B → C  
  
This behavior is similar to a **real-world queue (line)**.  
  
---  
  
## Queue Interface  
  
Since **Queue** is an interface, it cannot be instantiated directly.  
  
A concrete implementation must be used.  
    
Example:  
  
```  
Queue<Integer> queue = new LinkedList<>();  
```
  
---  
  
## Common Queue Implementations  
  
Java provides several classes that implement the Queue interface:  
  
- **LinkedList**    
- **PriorityQueue**    
- **ArrayDeque**  
  
Each implementation provides different ordering behavior.  
  
---  
  
## Basic Queue Operations  
  
The Queue interface provides several methods for insertion, removal, and inspection.  
  
### Insertion  
  
- add()  
- offer()  
  
### Removal  
  
- remove()  
- poll()  
  
### Inspection  
  
- peek()  
  
---  
  
## Method Behavior Differences  
  
| Method | Behavior if Queue is Full/Empty |
|-------|----------------------------------|
| add() | Throws exception if insertion fails |
| offer() | Returns false if insertion fails |
| remove() | Throws exception if queue is empty |
| poll() | Returns null if queue is empty |
| peek() | Returns head element without removing |
  
---  
  
## Time Complexity  
  
Typical queue operations are:  
  
- insertion: **O(1)**  
- removal: **O(1)**  
- peek: **O(1)**  
  
Actual complexity may vary depending on the specific queue implementation.  
  
---  
  
## When to Use Queues  
  
Queues are commonly used in:  
  
- Task scheduling  
- Breadth First Search (BFS)  
- Message processing systems  
- Producer–consumer problems  
- Request handling systems
