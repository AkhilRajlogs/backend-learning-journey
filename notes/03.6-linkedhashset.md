# LinkedHashSet in Java  
  
---  
  
## Overview  
  
LinkedHashSet is a Set implementation that **maintains insertion order**.  
  
It combines the characteristics of:  
  
- HashSet (hash table for fast lookup)  
- A doubly linked list (to maintain insertion order)  
  
Unlike HashSet, elements appear in the **same order in which they were inserted**.  
  
---  
  
## Key Characteristics  
  
- Stores only unique elements  
- Maintains insertion order  
- Allows one null element  
- Provides constant time performance for basic operations  
- Slightly slower than HashSet due to maintaining order  
  
---  
  
## Internal Structure  
  
LinkedHashSet internally uses **LinkedHashMap**.  
  
LinkedHashMap maintains:  
  
1. A **hash table** for fast lookup  
2. A **doubly linked list** to preserve insertion order  
  
Structure conceptually:  
  
    Hash Table + Doubly Linked List  
  
The linked list connects all entries in the order they were inserted.  
  
---  
  
## Example Behavior  
  
Insertion order is preserved.  
  
Example:  
  
    LinkedHashSet<String> set = new LinkedHashSet<>();  
  
    set.add("Java");  
    set.add("Python");  
    set.add("C++");  
  
Output:  
  
    [Java, Python, C++]  
  
Even though hashing is used internally, the order remains consistent.  
  
---  
  
## HashSet vs LinkedHashSet  
  
| Feature | HashSet | LinkedHashSet |  
|---------|---------|---------------|  
| Order | Not guaranteed | Maintains insertion order |  
| Performance | Slightly faster | Slightly slower |  
| Internal Structure | HashMap | LinkedHashMap |  
  
---  
  
## When to Use LinkedHashSet  
  
Use LinkedHashSet when:  
  
- You need **unique elements**  
- You want to **preserve insertion order**  
- You still want **fast lookup**  
  
Example use cases:  
  
- Removing duplicates while keeping original order  
- Maintaining ordered unique lists  
  
---  
  
## Important Note  
  
Like HashSet, LinkedHashSet relies on hashing.  
  
Therefore objects stored inside must properly override:  
  
    equals()  
    hashCode()  
  
Otherwise duplicate detection may fail.
