# PriorityQueue in Java
  
---  
  
## Overview  
  
PriorityQueue is a **Queue implementation** that orders elements according to their **priority** rather than insertion order.  
  
Unlike typical queues that follow the **FIFO (First-In-First-Out)** principle, PriorityQueue retrieves elements based on **priority ordering**.  
  
By default, elements are ordered according to their **natural ordering**, but a **custom Comparator** can also be provided.  
  
---  
  
## Key Characteristics  
  
- Orders elements based on priority  
- Does **not guarantee FIFO ordering**  
- Allows duplicate elements  
- Does **not allow null elements**  
- Internally implemented using a **Binary Heap**  
  
---  
  
## Internal Structure  
  
PriorityQueue is backed by a **Binary Heap** data structure.  
  
A binary heap maintains the **heap property** where the parent node is always less than or equal to its children (for a min-heap).  
  
Example heap structure:  
  
    10  
   /  \  
 20   30  
 /  
40  
  
The **smallest element remains at the root**, which allows quick access to the highest-priority element.  
  
---  
  
## Time Complexity  
  
Because heap operations require reordering elements:  
  
- add(): O(log n)  
- remove(): O(log n)  
- peek(): O(1)  
  
---  
  
## Example Behavior  
  
Example:  
  
    PriorityQueue<Integer> queue = new PriorityQueue<>();  
  
    queue.add(30);  
    queue.add(10);  
    queue.add(20);  
  
When removing elements:  
  
    10  
    20  
    30  
  
Elements are returned according to **priority order**, not insertion order.  
  
---  
  
## When to Use PriorityQueue  
  
PriorityQueue is useful when elements must be processed according to **priority** rather than insertion sequence.  
  
Common use cases:  
  
- Task scheduling systems  
- Dijkstra’s shortest path algorithm  
- Event simulation systems  
- Maintaining top-k elements  
  
---  
  
## Important Note  
  
PriorityQueue determines element ordering using:  
  
    Comparable  
    Comparator  
  
If elements do not implement Comparable and no Comparator is provided, a **ClassCastException** may occur.
