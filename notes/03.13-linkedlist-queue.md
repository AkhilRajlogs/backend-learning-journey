# LinkedList as Queue in Java
  
---  
  
## Overview  
  
LinkedList is a **doubly linked list implementation** in Java that implements the **List**, **Deque**, and **Queue** interfaces.  
  
Because it implements the Queue interface, LinkedList can be used directly as a **FIFO queue**.  
  
Unlike ArrayDeque which uses a **dynamic array**, LinkedList stores elements as **nodes connected by pointers**.  
  
---  
  
## Key Characteristics  
  
- Implemented using a **doubly linked list**  
- Supports **FIFO queue operations**  
- Allows **duplicate elements**  
- Allows **null elements**  
- Supports **insertion and removal from both ends**  
  
---  
  
## Internal Structure  
  
LinkedList internally maintains **nodes**.  
  
Each node contains:  
  
- the data element  
- reference to the **next node**  
- reference to the **previous node**  
  
Example structure:  
  
    null <- [10] <-> [20] <-> [30] -> null  
  
The list keeps references to:  
  
    head → first node  
    tail → last node  
  
Queue operations typically happen at the **head and tail**.  
  
---  
  
## Time Complexity  
  
Typical LinkedList queue operations:  
  
- add(): **O(1)**  
- offer(): **O(1)**  
- remove(): **O(1)**  
- poll(): **O(1)**  
- peek(): **O(1)**  
  
Since nodes are directly linked, insertion and removal at the ends are very efficient.  
  
---  
  
## Example Usage  
  
Example:  
  
    Queue<Integer> queue = new LinkedList<>();  
    
    queue.add(10);  
    queue.add(20);  
    queue.add(30);  
  
Removing elements:  
  
    10  
    20  
    30  
  
This demonstrates **FIFO ordering** when LinkedList is used as a queue.  
  
---  
  
## When to Use LinkedList as Queue  
  
LinkedList is useful when:  
  
- frequent **insertions and deletions** are required  
- memory overhead is not a major concern  
- null elements are allowed in the collection  
  
---  
  
## When Not to Use  
  
LinkedList may not be ideal when:  
  
- **high-performance queue operations** are required  
- memory efficiency is important  
- cache locality matters  
  
In many cases, **ArrayDeque performs better than LinkedList** for queue operations.  
  
---  
  
## Comparison with ArrayDeque  
  
Feature | LinkedList | ArrayDeque  
--- | --- | ---  
Internal Structure | Doubly Linked List | Dynamic Array  
Memory Overhead | Higher | Lower  
Null Elements | Allowed | Not Allowed  
Queue Performance | Good | Usually Faster  
  
---  
  
## Summary  
  
LinkedList can be used as a **Queue implementation in Java** because it implements the Queue interface.  
  
However, in most modern Java applications, **ArrayDeque is generally preferred** due to its better performance and lower memory overhead.
