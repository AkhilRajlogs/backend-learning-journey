# Map and HashMap Internals  
  
Map stores key-value pairs.  
  
### Characteristics:  
  
Keys are unique.  
  
Values can be duplicated.  
  
Map is part of the Java Collections Framework but does not extend the Collection interface.  
  
  ---
  
## HashMap  
    
### Internal Structure:  
  
HashMap is backed by an array of buckets.  
  
Each bucket stores a linked list.  
  
In Java 8+, if a bucket becomes too large, it may convert to a balanced tree (TreeNode) to improve performance.  
  
Uses hashing to determine bucket index.  
  
### Hashing Process  
  
key.hashCode() generates an integer hash.  
  
Bucket index is calculated using:  
  
index = hash & (capacity - 1)  
  
Capacity must be a power of 2 for this to work correctly.  
  
Using bitwise AND is faster than modulo operation.  
  
### Collision Handling  
  
When multiple keys map to same bucket:  
  
Nodes are stored using chaining (linked list).  
  
If key already exists → value updated.  
  
If not → new node inserted.  
  
Worst-case complexity becomes O(n) if all keys collide.  

Java 8 introduces treeification when a bucket contains more than 8 nodes and capacity is sufficiently large.  
This improves worst-case performance from O(n) to O(log n).  
  
### Load Factor  
  
Default load factor in Java = 0.75.  
  
When:  
  
size > capacity × loadFactor  
  
HashMap resizes.  
  
Resize doubles the capacity and redistributes elements.  
  
Resize Optimization (Java 8+ Insight)  
  
When capacity doubles:  
  
An element either:  
  
Stays at same index  
  
Moves to index + oldCapacity  
  
This avoids full rehash computation.  
  
## Time Complexity  
  
put(): O(1) average, O(n) worst case  
  
get(): O(1) average, O(n) worst case
