# HashSet in Java  
  
---  
  
## Overview  
  
HashSet is a collection that stores **unique elements**.  

It does not allow duplicates and does not guarantee insertion order.  
  
Internally, HashSet is implemented using a **HashMap**.  
  
Each element added to the HashSet is stored as a key in the underlying HashMap.  
  
---  

## Key Characteristics  
  
- Stores only unique elements  
- Does not maintain insertion order  
- Allows one null value  
- Not thread-safe  
- Provides constant time performance for basic operations  
  
---  
  
## Internal Implementation  
  
HashSet internally uses a HashMap.  
  
### Structure:  
  
Element → Stored as Key in HashMap    
Value → A constant dummy object  
  
### Example internal representation:  
  
    HashMap<E, Object>  

    Value = PRESENT (dummy object)  
  
So when we add an element:  
  
    set.add("Java")  
  
Internally becomes:  
  
    map.put("Java", PRESENT)  
  
---  
  
## Why Duplicates Are Not Allowed  
  
When inserting an element:  
  
1. Hash code of the element is calculated  
2. HashMap determines the bucket  
3. equals() is used to compare elements already present  
  
If an equal element exists → insertion is rejected.  
  
Therefore both methods must follow the contract:  
  
    hashCode()  
    equals()  
  
---  
  
## Time Complexity  
  
| Operation | Average Time |  
|----------|-------------|  
| Add | O(1) |  
| Remove | O(1) |  
| Contains | O(1) |  
    
Worst case can degrade to **O(n)** if many collisions occur.  
  
---  
  
## When to Use HashSet  
  
HashSet is useful when:  
  
- Only unique elements are required  
- Order of elements does not matter  
- Fast lookup is needed  
  
### Examples:  
  
- Removing duplicates from a list   
- Membership checking  
- Storing unique IDs or names  
  
---  
  
## HashSet vs ArrayList  
  
| Feature | ArrayList | HashSet |  
|--------|----------|---------|  
| Duplicates | Allowed | Not Allowed |  
| Order | Maintained | Not Guaranteed |  
| Lookup | O(n) | O(1) |  
  
---  
  
## HashSet vs LinkedHashSet  
  
### HashSet:  
  
- No ordering  
  
### LinkedHashSet:  
  
- Maintains insertion order  
  
#### LinkedHashSet internally combines:  
  
    Hash table + Linked list  
  
---  
  
## Important Note  
  
Because HashSet relies on hashing:  
  
Objects stored in HashSet should properly override:  
  
    equals()  
    hashCode()  
  
Otherwise duplicate objects may appear.
