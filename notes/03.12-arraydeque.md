# ArrayDeque in Java  
    
---  
  
## Overview  
  
ArrayDeque is a **resizable array implementation** of the **Deque interface** in Java.  
  
It can function as both:  
  
- a **Queue (FIFO)**  
- a **Stack (LIFO)**  
  
Unlike LinkedList, ArrayDeque is implemented using a **dynamic circular array**, which makes it **more efficient for most queue operations**.  
  
---  
  
## Key Characteristics  
  
- Implemented using a **resizable array**  
- Supports **insertion and removal from both ends**  
- Does **not allow null elements**  
- Faster than LinkedList for most queue operations  
- Automatically **resizes when capacity is exceeded**  
  
---  
  
## Internal Structure  
  
ArrayDeque internally maintains a **circular array buffer**.  
  
Two important indices are maintained:  
  
    head → index of the first element  
    tail → index where the next element will be inserted  
  
Example internal layout:  
  
    [ A | B | C | D | _ | _ | _ | _ ]  
      ↑           ↑  
     head        tail  
  
If the tail reaches the end of the array, it **wraps around to the beginning**, forming a circular structure.  
  
This design avoids the overhead of node allocations used in linked lists.  
  
---  
  
## Time Complexity  
  
Typical operations in ArrayDeque are highly efficient:  
  
- add(): **O(1) amortized**  
- offer(): **O(1) amortized**  
- remove(): **O(1)**  
- poll(): **O(1)**  
- peek(): **O(1)**  
  
Because the structure is array-based, it avoids pointer overhead.  
  
---  
  
## Example Usage  
  
Example:  
  
    Queue<Integer> queue = new ArrayDeque<>();  
    
    queue.add(10);  
    queue.add(20);  
    queue.add(30);  
  
When removing elements:  
  
    10  
    20  
    30  
  
This demonstrates **FIFO behavior when used as a queue**.  
  
---  
  
## When to Use ArrayDeque  
  
ArrayDeque is recommended when:  
  
- You need a **fast Queue implementation**  
- You want **better performance than LinkedList**  
- You want a structure that can behave as both **queue and stack**  
  
In many cases, **ArrayDeque is preferred over LinkedList for queue operations**.  
  
---  
  
## When Not to Use  
  
ArrayDeque is not suitable when:  
  
- You need **priority-based ordering (use PriorityQueue)**  
- You need **thread-safe queue implementations**  
- You need **null elements in the collection**  
  
---  
  
## Comparison with LinkedList  
  
Feature | ArrayDeque | LinkedList  
--- | --- | ---  
Internal Structure | Dynamic Array | Doubly Linked List  
Memory Overhead | Lower | Higher  
Queue Performance | Faster | Slightly Slower  
Null Elements | Not Allowed | Allowed  
  
---  
  
## Summary  
  
ArrayDeque is a **high-performance queue and deque implementation in Java**.  
  
It provides:  
  
- fast insertion and removal  
- efficient memory usage  
- flexible usage as both **stack and queue**  
  
For most queue use cases in Java, **ArrayDeque is preferred over LinkedList**.
